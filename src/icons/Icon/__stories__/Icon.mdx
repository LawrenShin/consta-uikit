# Icon  

Иконки нужны для ускорения восприятия информации. Так происходит потому, что у людей образное мышление. Если мы видим образ, нам проще понимать суть.  
В интерфейсах большинство понятий — абстракции, поэтому объяснить их одними пиктограммами бывает сложно.

Несмотря на то, что за некоторыми действиями и паттернами уже закрепились определенные пиктограммы (например ```☰``` — меню),  
в спорных ситуациях лучше применять паттерн иконка + текст/заголовок.
  
Если вы используете иконку без поддержки текста, нужно особенно внимательно следить за метафорой, которую несет иконка.  
Взглянув на иконку, нужно однозначно понимать какой посыл она несет. 

## Свойства

| Свойство     | Тип                                                                               | По умолчанию | Описание                                  |
| ------------ | --------------------------------------------------------------------------------- | ------------ | ----------------------------------------- |
| `size?`      | `'xs', 's', 'm'`                                                                  | `'m'`        | Размер                                    |
| `view?`      | `'alert', 'brand', 'ghost', 'link', 'primary', 'secondary', 'success', 'warning'` | -            | Вид                                       |
| `className?` | `string`                                                                          | -            | Дополнительный CSS-класс                  |
| `ref?`       | `React.Ref<HTMLSpanElement>`                                                      | -            | Ссылка на корневой DOM-элемент компонента |

## Пример использования в кнопке

```tsx
import { IconCancel } from '@consta/uikit/IconCancel'

const buttonWithIcon = () => {
  return (
    <Button
      label="Закрыть"
      iconLeft={IconCancel}
      onlyIcon
    />
  )}

export default buttonWithIcon
```

## Создание своих иконок

Компоненты **CONSTA** принимают иконки определенного типа. Чтобы ваши иконки совпадали этому типу,
воспользуйтесть функцией `createIcon`:

```tsx
import { createIcon } from '@consta/uikit/createIcon';

import IconAddSizeM from './IconAdd_size_m';
import IconAddSizeS from './IconAdd_size_s';
import IconAddSizeXs from './IconAdd_size_xs';

export const IconAdd = createIcon({
  m: IconAddSizeM,
  s: IconAddSizeS,
  xs: IconAddSizeXs,
  name: 'IconAdd',
});
```

Требование к SVG:

- Должны отсутствовать атрибуты: `fill`, `width`, `height`,

#### Если есть только один размер иконки

Ничего страшного в том, если у вас нет разных иконок под каждый размер,
просто укажите одну иконку во всех размерах:

```tsx
import { createIcon } from '@consta/uikit/createIcon';

import IconAddSizeM from './IconAdd_size_m';

export const IconAdd = createIcon({
  m: IconAddSizeM,
  s: IconAddSizeM,
  xs: IconAddSizeM,
  name: 'IconAdd',
});
```

#### Если иконки автоматически собираются из SVG в Компоненты

Если ваша сборка уже настроена так, что иконки из SVG собираются в Компоненты,
вы можете добавить в конфигурацию `createIcon`. Пример webpack со сборкой иконок через SVGR:

```jsx
{
    test: /\.svg$/,
    use: [
        {
            loader: '@svgr/webpack',
            options: {
                template: (
                    { template },
                    opts,
                    { imports, componentName, props, jsx, exports },
                ) => {
                    return template.ast`
                        ${imports}
                        import { createIcon } from '@consta/uikit/createIcon';

                        const SvgComponent = (${props}) => {
                          return ${jsx};
                        };

                        const ${componentName} = createIcon({
                          m: SvgComponent,
                          s: SvgComponent,
                          xs: SvgComponent,
                          name: '${componentName}',
                        });

                        export default ${componentName};
                  `;
                },
            },
        },
    ],
},
```
